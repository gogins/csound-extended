Csound HTML5 Demonstration 02

Introduction

The purpose of this demonstration is to show how to embed a Csound piece into a Web page. This demonstration builds on the previous one by adding a slider to control each global channel in the Csound orchestra, and using jQuery to simplify setting up event handling for the sliders. The instruments in this piece are sensitive to control, and playing with the sliders during performance can change the character of the sounds and of the piece to a significant degree. You can play the piece, stop it from playing, view and edit its Csound code, and see Csound's diagnostic messages. This piece will play on csound.node. A thorough explanation of the steps required to create this piece follows.

How It Is Done

This demonstration is simplified. All unnecessary HTML elements and JavaScript function calls have been omitted. This Web page uses only a subset of the Csound API that has the same funtions, names, and signatures in csound.node, Csound for Android, and CsoundQt.

Comments on this page or in the page source explain what each element is for and why it is written the way it is. You can view the code and comments by right-clicking on this page and selecting "View page source".

Embedding Csound

With Csound for Android and CsoundQt, the front end has already embedded its running instance of Csound into the Javascript context of the Web page. With csound.node, as here, you can use node's require module system to include csound, and you also need to set the output option (see below on Starting and Stopping Csound for the code). As far as any JavaScript code on the Web page is concerned, Csound is just an already existing JavaScript object named "csound", with its own functions and callbacks, which represent important parts of the Csound API.

It is good practice to place all <script> elements at the very end of the HTML <body> element, so that the other elements have already been created before any scripts try to access them, and that is what I have done here.

Embedding the Csound Score and Orchestra

There are many ways to get a Csound piece into Csound. But in order to make the Csound orchestra and score editable, they must appear in <textarea> elements on the Web page. These can be smaller than the text they contain, yet have scrollbars for accessing all of the text. These textareas must be named, so that we can use JavaScript to retrieve the orchestra and score and pass them to Csound. The code for a <textarea> containing a Csound orchestra looks like this:

<textarea id="orc" rows=24 cols=80> opcode hertz2midinn, i, i ihertz xin print ihertz ; m = 12*log2(fm/440 Hz) + 69 ilog2 = log(2) imidinn = 12 * (log(ihertz / 440) / ilog2) + 69 print imidinn xout imidinn endop instr 101 insno = p1 istart = p2 iduration = p3 ihertz = p4 ivelocity = p5 ipan = p6 ikey hertz2midinn ihertz event_i "i", "Droner", 0, iduration, ikey, ivelocity, 0, ipan endin (...remainder of orchestra...) </textarea>

The code for a <textarea> containing a Csound score works the same way.

Also note, Csound can perform scores using either csound.compileOrc(orc_text) followed by csound.readScore(sco_text), or by simply calling csound.playCsd(filename). If readScore is used, the score must not contain any "t", "s", or "e" statements, and all score times must be absolute times in seconds from the beginning of the performance. If playCsd is used, any and all standard score statements may be used.

Starting and Stopping Csound

It is convenient to provide buttons for starting and stopping Csound, rather than simply having it start playing the moment the page is loaded. The JavaScript code for handling the button clicks must start Csound, send the orchestra to Csound, and send the score, in order to perform the piece. The buttons and code for the Play and Stop buttons have been modified from the first demonstration to use jQuery, see below.

Using Sliders to Control Csound

The HTML <input> element has a variety of types, and the range type creates a slider control. We use an HTML table to lay out the sliders, labels, and numeric display fields in a grid. We use a number of strategies to simplify, as far as possible, the use of such sliders to control Csound instruments.

First, we adopt the principle that the Csound instruments themselves are encapsulated into independent modules that can be mixed, matched, and re-arranged. There is not space here to discuss this in detail, but in essence, each instrument is contained in a self-sufficient block of code. All real-time controls are global variables, initialized just above the actual "instr" block. All function tables, also, are initialized just above the "instr" block. A consistent naming pattern is used to avoid name clashes, e.g., "gk_" + instrument name + "_" + control name, to control the amount of delay in the global reverberation effect. Instruments do not send sound directly to the output, but use audio channels to communicate with effects, which in turn send sound to the output. The result is that it is easy to re-order the instruments, or to include instruments from other orchestras, simply by cutting and pasting, or with #include statements in the orchestra. Here is the code from one instrument:

gi_Buzzer_overlap init 20 gk_Buzzer_harmonics init 2 gi_Buzzer_sine ftgen 0, 0, 65536, 10, 1 instr Buzzer insno = p1 istart = p2 iduration = p3 ikey = p4 ivelocity = p5 iphase = p6 ipan = p7 iamp = ampdb(ivelocity) * 4 iattack = gi_Buzzer_overlap idecay = gi_Buzzer_overlap isustain = p3 - gi_Buzzer_overlap p3 = iattack + isustain + idecay kenvelope transeg 0.0, iattack / 2.0, 1.5, iamp / 2.0, iattack / 2.0, -1.5, iamp, isustain, 0.0, iamp, idecay / 2.0, 1.5, iamp / 2.0, idecay / 2.0, -1.5, 0 ihertz = cpsmidinn(ikey) asignal buzz kenvelope, ihertz, gk_Buzzer_harmonics, gi_Buzzer_sine asignal = asignal * 3 aleft, aright pan2 asignal, ipan adamping linseg 0, 0.03, 1, p3 - 0.1, 1, 0.07, 0 aleft = adamping * aleft aright = adamping * aright chnmix aleft, "out_left" chnmix aright, "out_right" prints "instr %4d t %9.4f d %9.4f k %9.4f v %9.4f p %9.4f\n", p1, p2, p3, p4, p5, p7 endin

The "effects" instruments, as well as a global "Controls" instrument, are "always on" and they are scheduled in the orchestra code, not from the score. This makes it much easier to generate scores programmatically, and comes in handy later.

scoreline_i "i \"Reverb\" 0 -1" scoreline_i "i \"MasterOutput\" 0 -1" scoreline_i "i \"Controls\" 0 -1" scoreline_i "i \"Clear\" 0 -1"

The "Controls" instrument simply has a chnget opcode for each control channel. This is much more efficient than putting chnget into each instrument. I have added buttons for saving and restoring the values of all controls; saved values persist between sessions. Something like this is essential for actual composing or performance.

Second, the jQuery JavaScript utility library is used to simplify the event handling, and to get the values of the sliders into Csound. "Event bubbling" is used to send events from all sliders to one event handler. We make sure that the HTML identifiers of all the sliders are exactly the same as the corresponding Csound control channels, and use a naming pattern to link output fields to sliders, also. This makes writing a single event handler for all sliders very easy. Similarly, jQuery simplifies saving and restoring slider values to and from local storage. In jQuery, $('#name') returns the DOM element with id 'name', and $('element') returns all elements of type 'element'. The entire event handling code is:

<script> var csound = require('csound'); </script> <script src="https://ajax.googleapis.com/ajax/libs/jquery/1.11.3/jquery.min.js"> </script> <script> <script> $(document).ready(function() { $('#play').on('click', function() { csound.setOption('-odac') csound.compileOrc(document.getElementById('orc').value); csound.readScore(document.getElementById('sco').value); $('#restore').trigger('click'); csound.perform(); }); $('#stop').on('click', function() { csound.stop(); }); $('input').on('input', function(event) { var slider_value = parseFloat(event.target.value); csound.setControlChannel(event.target.id, slider_value); var output_selector = '#' + event.target.id + '_output'; $(output_selector).val(slider_value); }); $('#save').on('click', function() { $('.persistent-element').each(function() { localStorage.setItem(this.id, this.value); }); }); $('#restore').on('click', function() { $('.persistent-element').each(function() { this.value = localStorage.getItem(this.id); csound.setControlChannel(this.id, this.value); var output_selector = '#' + this.id + '_output'; $(output_selector).val(this.value); }); }); $(window).load(function() { $('#restore').trigger('click'); }); }); </script>

Handling Csound's Errors, Warnings, and Messages

On csound.node, Csound sends all of its runtime messages to the JavaScript console.
